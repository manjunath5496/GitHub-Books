
<p><strong>GitHub, Inc.</strong>&nbsp;is a US-based global company that provides&nbsp;<a title="Internet hosting service" href="https://en.wikipedia.org/wiki/Internet_hosting_service">hosting</a>&nbsp;for&nbsp;<a title="Software development" href="https://en.wikipedia.org/wiki/Software_development">software development</a>&nbsp;<a title="Version control" href="https://en.wikipedia.org/wiki/Version_control">version control</a>&nbsp;using&nbsp;<a title="Git" href="https://en.wikipedia.org/wiki/Git">Git</a>. It is a subsidiary of&nbsp;<a title="" href="https://en.wikipedia.org/wiki/Microsoft">Microsoft</a>, which acquired the company in 2018 for&nbsp;<a class="mw-redirect" title="US$" href="https://en.wikipedia.org/wiki/US$">US$</a>7.5 billion.&nbsp;It offers the&nbsp;<a title="Distributed version control" href="https://en.wikipedia.org/wiki/Distributed_version_control">distributed version control</a>&nbsp;and&nbsp;<a class="mw-redirect" title="Source code management" href="https://en.wikipedia.org/wiki/Source_code_management">source code management</a>&nbsp;(SCM) functionality of Git, plus its own features. It provides&nbsp;<a title="Access control" href="https://en.wikipedia.org/wiki/Access_control">access control</a>&nbsp;and several collaboration features such as&nbsp;<a title="Bug tracking system" href="https://en.wikipedia.org/wiki/Bug_tracking_system">bug tracking</a>,&nbsp;<a title="Software feature" href="https://en.wikipedia.org/wiki/Software_feature">feature</a>&nbsp;requests,&nbsp;<a title="Task management" href="https://en.wikipedia.org/wiki/Task_management">task management</a>, and&nbsp;<a title="Wiki" href="https://en.wikipedia.org/wiki/Wiki">wikis</a>&nbsp;for every project.</p>
<p>GitHub offers plans free of charge, and professional and enterprise accounts.&nbsp;Free GitHub accounts are commonly used to host&nbsp;<a title="Open-source software" href="https://en.wikipedia.org/wiki/Open-source_software">open source</a>&nbsp;projects.&nbsp;As of January 2019, GitHub offers unlimited private repositories to all plans, including free accounts.&nbsp;As of January 2020, GitHub reports having over 40&nbsp;million users<sup id="cite_ref-8" class="reference"><a href="https://en.wikipedia.org/wiki/GitHub#cite_note-8">[8]</a></sup>&nbsp;and more than 100&nbsp;million&nbsp;<a title="Repository (version control)" href="https://en.wikipedia.org/wiki/Repository_(version_control)">repositories</a>&nbsp;(including at least 28&nbsp;million public repositories),&nbsp;making it the largest host of&nbsp;<a title="Source code" href="https://en.wikipedia.org/wiki/Source_code">source code</a>&nbsp;in the world.</p>

<h2><span id="History" class="mw-headline">History</span></h2>
<div class="thumb tright">&nbsp;</div>
<p>GitHub was developed by&nbsp;<a title="Chris Wanstrath" href="https://en.wikipedia.org/wiki/Chris_Wanstrath">Chris Wanstrath</a>,&nbsp;<a title="P. J. Hyett" href="https://en.wikipedia.org/wiki/P._J._Hyett">P. J. Hyett</a>,&nbsp;<a title="Tom Preston-Werner" href="https://en.wikipedia.org/wiki/Tom_Preston-Werner">Tom Preston-Werner</a>&nbsp;and Scott Chacon using&nbsp;<a title="Ruby on Rails" href="https://en.wikipedia.org/wiki/Ruby_on_Rails">Ruby on Rails</a>, and started in February 2008. The company, GitHub, Inc., has existed since 2007 and is located in San Francisco.</p>
<div class="thumb tright">&nbsp;</div>
<p>On February 24, 2009, GitHub team members announced, in a talk at&nbsp;<a title="Yahoo!" href="https://en.wikipedia.org/wiki/Yahoo!">Yahoo!</a>&nbsp;headquarters, that within the first year of being online, GitHub had accumulated over 46,000 public repositories, 17,000 of which were formed in the previous month alone. At that time, about 6,200 repositories had been&nbsp;<a title="Fork (software development)" href="https://en.wikipedia.org/wiki/Fork_(software_development)">forked</a>&nbsp;at least once and 4,600 had been merged.</p>
<p>On July 5, 2009, GitHub announced that the site was being harnessed by over 100,000 users. On July 27, 2009, in another talk delivered at&nbsp;<a title="Yahoo!" href="https://en.wikipedia.org/wiki/Yahoo!">Yahoo!</a>, Preston-Werner announced that GitHub had grown to host 90,000 unique public repositories, 12,000 having been forked at least once, for a total of 135,000 repositories.</p>
<p>On July 25, 2010, GitHub announced that it was hosting 1 million repositories.&nbsp;On April 20, 2011, GitHub announced that it was hosting 2 million repositories.</p>
<p>On June 2, 2011,&nbsp;<a class="mw-redirect" title="ReadWriteWeb" href="https://en.wikipedia.org/wiki/ReadWriteWeb">ReadWriteWeb</a>&nbsp;reported that GitHub had surpassed&nbsp;<a title="SourceForge" href="https://en.wikipedia.org/wiki/SourceForge">SourceForge</a>&nbsp;and&nbsp;<a class="mw-redirect" title="Google Code" href="https://en.wikipedia.org/wiki/Google_Code">Google Code</a>&nbsp;in total number of commits for the period of January to May 2011.</p>
<p>On July 9, 2012, GitHub raised $100 million in funding from Andreessen Horowitz with $750 million valuation.&nbsp;<a class="mw-redirect" title="Peter Levine (venture capitalist)" href="https://en.wikipedia.org/wiki/Peter_Levine_(venture_capitalist)">Peter Levine</a>, general partner at GitHub investor&nbsp;<a title="Andreessen Horowitz" href="https://en.wikipedia.org/wiki/Andreessen_Horowitz">Andreessen Horowitz</a>, stated that GitHub had been growing revenue at 300% annually since 2008 "profitably nearly the entire way".</p>
<p>On January 16, 2013, GitHub announced it had passed the 3 million users mark and was then hosting more than 5&nbsp;million repositories.&nbsp;On December 23, 2013, GitHub announced it had reached 10 million repositories.</p>
<p>In June 2015, GitHub opened an office in Japan that is its first office outside of the U.S.</p>
<p>On July 29, 2015, GitHub announced it had raised $250&nbsp;million in funding in a round led by&nbsp;<a title="Sequoia Capital" href="https://en.wikipedia.org/wiki/Sequoia_Capital">Sequoia Capital</a>. Other investors of this round include Andreessen Horowitz, Thrive Capital, and IVP (Institutional Venture Partners).&nbsp;&nbsp;The round valued the company at approximately $2&nbsp;billion.</p>
<p>In 2016, GitHub was ranked No. 14 on the&nbsp;<em><a title="Forbes" href="https://en.wikipedia.org/wiki/Forbes">Forbes</a></em>&nbsp;Cloud 100 list.</p>
<p>On February 28, 2018, GitHub fell victim to the second largest&nbsp;<a class="mw-redirect" title="Distributed denial-of-service" href="https://en.wikipedia.org/wiki/Distributed_denial-of-service">distributed denial-of-service</a>&nbsp;(DDoS) attack in history, with incoming traffic reaching a peak of about 1.35 terabits per second.</p>
<p>On June 4, 2018,&nbsp;<a title="Microsoft" href="https://en.wikipedia.org/wiki/Microsoft">Microsoft</a>&nbsp;announced it had reached an agreement to acquire GitHub for $7.5 billion.&nbsp;The purchase closed on October 26, 2018.</p>
<p>On June 19, 2018, GitHub expanded its GitHub Education by offering free education bundles to all schools.</p>
<p>On February 12, 2020, GitHub launched in India under the name GitHub India Private Limited.</p>

<h3><span id="Acquisition_by_Microsoft" class="mw-headline">Acquisition by Microsoft</span></h3>
<p>On June 4, 2018,&nbsp;<a title="Microsoft" href="https://en.wikipedia.org/wiki/Microsoft">Microsoft</a>&nbsp;announced its intent to acquire GitHub for US$7.5 billion, and the deal closed on October 26, 2018.&nbsp;According to Microsoft, GitHub will continue to operate independently as a community, platform and business. Under Microsoft, the service will be led by&nbsp;<a title="Xamarin" href="https://en.wikipedia.org/wiki/Xamarin">Xamarin</a>'s&nbsp;<a title="Nat Friedman" href="https://en.wikipedia.org/wiki/Nat_Friedman">Nat Friedman</a>, reporting to&nbsp;<a title="Scott Guthrie" href="https://en.wikipedia.org/wiki/Scott_Guthrie">Scott Guthrie</a>, executive vice president of Microsoft Cloud and AI. Current CEO Chris Wanstrath will be retained as a "technical fellow", also reporting to Guthrie. Microsoft had become a significant user of GitHub, using it to host open source projects and development tools such as&nbsp;<a title="Chakra (JavaScript engine)" href="https://en.wikipedia.org/wiki/Chakra_(JavaScript_engine)">Chakra Core</a>,&nbsp;<a title="PowerShell" href="https://en.wikipedia.org/wiki/PowerShell">PowerShell</a>,&nbsp;<a title="Visual Studio Code" href="https://en.wikipedia.org/wiki/Visual_Studio_Code">Visual Studio Code</a>, and&nbsp;<a title="Windows Terminal" href="https://en.wikipedia.org/wiki/Windows_Terminal">Windows Terminal</a>&nbsp;and has backed other open source projects such as&nbsp;<a title="Linux" href="https://en.wikipedia.org/wiki/Linux">Linux</a>, and developed&nbsp;<a title="Virtual File System for Git" href="https://en.wikipedia.org/wiki/Virtual_File_System_for_Git">Virtual File System for Git</a>&nbsp;(VFS for Git; formerly Git Virtual File System or GVFS)&mdash;a Git extension for managing large-scale repositories (and itself has been adopted by GitHub).</p>
<p>There have been concerns from developers Kyle Simpson,&nbsp;<a title="JavaScript" href="https://en.wikipedia.org/wiki/JavaScript">JavaScript</a>&nbsp;trainer and author, and Rafael Laguna, CEO at&nbsp;<a title="Open-Xchange" href="https://en.wikipedia.org/wiki/Open-Xchange">Open-Xchange</a>&nbsp;over Microsoft's purchase, citing uneasiness over Microsoft's handling of previous acquisitions, such as&nbsp;<a title="Microsoft Mobile" href="https://en.wikipedia.org/wiki/Microsoft_Mobile">Nokia's mobile business</a>&nbsp;or&nbsp;<a title="Skype" href="https://en.wikipedia.org/wiki/Skype">Skype</a>.</p>
<p>Some<sup class="noprint Inline-Template">[<em><a title="Wikipedia:Manual of Style/Words to watch" href="https://en.wikipedia.org/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions"><span title="The material near this tag possibly uses too-vague attribution or weasel words. (August 2018)">who?</span></a></em>]</sup>&nbsp;saw this as a culmination of Microsoft's recent changes in business strategy under CEO&nbsp;<a title="Satya Nadella" href="https://en.wikipedia.org/wiki/Satya_Nadella">Satya Nadella</a>, which has seen a larger focus on the sale of&nbsp;<a title="Cloud computing" href="https://en.wikipedia.org/wiki/Cloud_computing">cloud computing</a>&nbsp;services as its main line of business, alongside development of and contributions to open source software (such as&nbsp;<a title="Linux" href="https://en.wikipedia.org/wiki/Linux">Linux</a>), as opposed to the&nbsp;<a title="Microsoft Windows" href="https://en.wikipedia.org/wiki/Microsoft_Windows">Microsoft Windows</a>&nbsp;operating system.&nbsp;<em><a title="Harvard Business Review" href="https://en.wikipedia.org/wiki/Harvard_Business_Review">Harvard Business Review</a></em>&nbsp;argued that Microsoft was intending to acquire GitHub to get access to its user base, so it can be used as a&nbsp;<a title="Loss leader" href="https://en.wikipedia.org/wiki/Loss_leader">loss leader</a>&nbsp;to encourage use of its other development products and services.</p>
<p>Concerns over the sale bolstered interest in competitors:&nbsp;<a title="Bitbucket" href="https://en.wikipedia.org/wiki/Bitbucket">Bitbucket</a>&nbsp;(owned by&nbsp;<a title="Atlassian" href="https://en.wikipedia.org/wiki/Atlassian">Atlassian</a>),&nbsp;<a title="GitLab" href="https://en.wikipedia.org/wiki/GitLab">GitLab</a>&nbsp;(a&nbsp;<a class="mw-redirect" title="Commercial open source" href="https://en.wikipedia.org/wiki/Commercial_open_source">commercial open source</a>&nbsp;product that also runs a hosted service version) and&nbsp;<a title="SourceForge" href="https://en.wikipedia.org/wiki/SourceForge">SourceForge</a>&nbsp;(owned by BIZX, LLC) reported that they had seen spikes in new users intending to migrate projects from GitHub to their respective services.</p>
<p>In March 2020, GitHub announced that they are acquiring&nbsp;<a title="Npm (software)" href="https://en.wikipedia.org/wiki/Npm_(software)">npm</a>, a JavaScript packaging vendor, for an undisclosed sum of money.</p>
<h2><span id="Company_affairs" class="mw-headline">Company affairs</span></h2>
<h3><span id="Organizational_structure" class="mw-headline">Organizational structure</span></h3>
<p>GitHub, Inc. was originally a&nbsp;<a title="Flat organization" href="https://en.wikipedia.org/wiki/Flat_organization">flat organization</a>&nbsp;with no middle managers; in other words, "everyone is a manager" (<a class="mw-redirect" title="Workers' self-management" href="https://en.wikipedia.org/wiki/Workers%27_self-management">self-management</a>).&nbsp;Employees could choose to work on projects that interested them (<a title="Open allocation" href="https://en.wikipedia.org/wiki/Open_allocation">open allocation</a>), but salaries were set by the chief executive.</p>
<p>In 2014, GitHub, Inc. introduced a layer of middle management.</p>
<h3><span id="Finance" class="mw-headline">Finance</span></h3>
<p>GitHub.com was a&nbsp;<a class="mw-redirect" title="Bootstrapped" href="https://en.wikipedia.org/wiki/Bootstrapped">bootstrapped</a>&nbsp;<a title="Startup company" href="https://en.wikipedia.org/wiki/Startup_company">start-up business</a>, which in its first years provided enough revenue to be funded solely by its three founders and start taking on employees.&nbsp;In July 2012, four years after the company was founded,&nbsp;<a title="Andreessen Horowitz" href="https://en.wikipedia.org/wiki/Andreessen_Horowitz">Andreessen Horowitz</a>&nbsp;invested $100&nbsp;million in&nbsp;<a title="Venture capital" href="https://en.wikipedia.org/wiki/Venture_capital">venture capital</a>.&nbsp;In July 2015 GitHub raised another $250&nbsp;million of venture capital in a&nbsp;<a class="mw-redirect" title="Series B" href="https://en.wikipedia.org/wiki/Series_B">series B round</a>. Investors were&nbsp;<a title="Sequoia Capital" href="https://en.wikipedia.org/wiki/Sequoia_Capital">Sequoia Capital</a>,&nbsp;<a title="Andreessen Horowitz" href="https://en.wikipedia.org/wiki/Andreessen_Horowitz">Andreessen Horowitz</a>,&nbsp;<a class="mw-redirect" title="Thrive Capital" href="https://en.wikipedia.org/wiki/Thrive_Capital">Thrive Capital</a>&nbsp;and other venture capital funds.&nbsp;As of August 2016, GitHub was making $140&nbsp;million in Annual Recurring Revenue.</p>
<h3><span id="Mascot" class="mw-headline">Mascot</span></h3>
<p>GitHub's&nbsp;<a title="Mascot" href="https://en.wikipedia.org/wiki/Mascot">mascot</a>&nbsp;is an&nbsp;<a class="mw-redirect" title="Anthropomorphized" href="https://en.wikipedia.org/wiki/Anthropomorphized">anthropomorphized</a>&nbsp;"octocat" with five&nbsp;<a title="Cephalopod limb" href="https://en.wikipedia.org/wiki/Cephalopod_limb">octopus-like arms</a>.</p>
<p>The character was created by graphic designer&nbsp;<a title="Simon Oxley" href="https://en.wikipedia.org/wiki/Simon_Oxley">Simon Oxley</a>&nbsp;as&nbsp;<a title="Clip art" href="https://en.wikipedia.org/wiki/Clip_art">clip art</a>&nbsp;to sell on&nbsp;<a title="IStock" href="https://en.wikipedia.org/wiki/IStock">iStock</a>,&nbsp;a website that enables designers to market&nbsp;<a title="Royalty-free" href="https://en.wikipedia.org/wiki/Royalty-free">royalty-free</a>&nbsp;<a title="Digital image" href="https://en.wikipedia.org/wiki/Digital_image">digital images</a>.</p>
<p>GitHub became interested in Oxley's work after Twitter selected a bird that he designed for their own logo.&nbsp;The illustration GitHub chose was a character that Oxley had named Octopuss.&nbsp;Since GitHub wanted Octopuss for their logo (a use that the iStock license disallows), they negotiated with Oxley to buy exclusive rights to the image.</p>
<p>GitHub renamed Octopuss to Octocat,&nbsp;and trademarked the character along with the new name.&nbsp;Later, GitHub hired illustrator Cameron McEfee to adapt Octocat for different purposes on the website and promotional materials; McEfee and various GitHub users have since created hundreds of variations of the character, which are available on&nbsp;<a class="external text" href="https://octodex.github.com/" rel="nofollow">The Octodex</a>.</p>
<h2><span id="Services" class="mw-headline">Services</span></h2>
<h3><span id="GitHub" class="mw-headline">GitHub</span></h3>
<p>Development of the GitHub platform began on October 19, 2007.&nbsp;The site was launched in April 2008 by Tom Preston-Werner, Chris Wanstrath, P. J. Hyett and Scott Chacon after it had been made available for a few months prior as a beta release.</p>
<p>Projects on GitHub can be accessed and manipulated using the standard Git command-line interface and all of the standard Git commands work with it. GitHub also allows registered and unregistered users to browse public repositories on the site. Multiple desktop clients and Git&nbsp;<a title="Plug-in (computing)" href="https://en.wikipedia.org/wiki/Plug-in_(computing)">plugins</a>&nbsp;have also been created by GitHub and other third parties that integrate with the platform.</p>
<p>The site provides&nbsp;<a class="mw-redirect" title="Social networking" href="https://en.wikipedia.org/wiki/Social_networking">social networking</a>-like functions such as feeds, followers, wikis (using&nbsp;<a title="Wiki software" href="https://en.wikipedia.org/wiki/Wiki_software">wiki software</a>&nbsp;called&nbsp;<a title="Gollum Wiki" href="https://en.wikipedia.org/wiki/Gollum_Wiki">Gollum</a>) and a&nbsp;<a title="Collaboration graph" href="https://en.wikipedia.org/wiki/Collaboration_graph">social network graph</a>&nbsp;to display how developers work on their versions ("<a title="Fork (software development)" href="https://en.wikipedia.org/wiki/Fork_(software_development)">forks</a>") of a repository and what fork (and branch within that fork) is newest.</p>
<p>A user must create an account in order to contribute content to the site, but public repositories can be browsed and downloaded by anyone. With a registered user account, users are able to have discussions, manage repositories, submit contributions to others' repositories, and&nbsp;<a title="Code review" href="https://en.wikipedia.org/wiki/Code_review">review changes to code</a>. GitHub began offering unlimited private repositories at no cost in January 2019 (limited to three contributors per project). Previously, only public repositories were free.</p>
<p>The fundamental software that underpins GitHub is&nbsp;<a title="Git" href="https://en.wikipedia.org/wiki/Git">Git</a>&nbsp;itself, written by&nbsp;<a title="Linus Torvalds" href="https://en.wikipedia.org/wiki/Linus_Torvalds">Linus Torvalds</a>, creator of Linux. The additional software that provides the GitHub user interface was written using&nbsp;<a title="Ruby on Rails" href="https://en.wikipedia.org/wiki/Ruby_on_Rails">Ruby on Rails</a>&nbsp;and&nbsp;<a title="Erlang (programming language)" href="https://en.wikipedia.org/wiki/Erlang_(programming_language)">Erlang</a>&nbsp;by GitHub, Inc. developers Wanstrath,&nbsp;Hyett, and Preston-Werner.</p>

<h4><span id="Licensing_of_repositories" class="mw-headline">Licensing of repositories</span></h4>
<p>GitHub's Terms of Service do not require public software projects hosted on GitHub to meet the&nbsp;<a class="mw-redirect" title="Open Source Definition" href="https://en.wikipedia.org/wiki/Open_Source_Definition">Open Source Definition</a>. For that reason, it is essential for users and developers intending to use a piece of software found on GitHub to read the&nbsp;<a title="Software license" href="https://en.wikipedia.org/wiki/Software_license">software license</a>&nbsp;in the repository (usually found in a top-level file called "LICENSE", "LICENSE.txt", or similar) to determine if it meets their needs<sup class="noprint Inline-Template Template-Fact">[<em><a title="Wikipedia:Citation needed" href="https://en.wikipedia.org/wiki/Wikipedia:Citation_needed"><span title="This claim needs references to reliable sources. (July 2016)">citation needed</span></a></em>]</sup>. The&nbsp;<a class="mw-redirect" title="Terms of Service" href="https://en.wikipedia.org/wiki/Terms_of_Service">Terms of Service</a>&nbsp;state, "By setting your repositories to be viewed publicly, you agree to allow others to view and fork your repositories."</p>
<h3><span id="GitHub_Enterprise" class="mw-headline">GitHub Enterprise</span></h3>
<p>GitHub Enterprise is a self-managed version of GitHub with similar functionality. It can be run on an organisation's own hardware or on a cloud provider, and it has been available since November 2011.</p>
<h3><span id="GitHub_Pages" class="mw-headline">GitHub Pages</span></h3>
<p><strong>GitHub Pages</strong>&nbsp;is a&nbsp;<a title="Static web page" href="https://en.wikipedia.org/wiki/Static_web_page">static</a>&nbsp;<a title="Web hosting service" href="https://en.wikipedia.org/wiki/Web_hosting_service">web hosting service</a>&nbsp;offered by GitHub since 2008 to GitHub users for hosting user&nbsp;<a title="Blog" href="https://en.wikipedia.org/wiki/Blog">blogs</a>, project documentation,&nbsp;or even whole books created as a page.</p>
<p>All GitHub Pages content is stored in Git repository, either as files served to visitors verbatim or in&nbsp;<a class="mw-redirect" title="GitHub Flavored Markdown" href="https://en.wikipedia.org/wiki/GitHub_Flavored_Markdown">Markdown</a>&nbsp;format. GitHub is seamlessly integrated with&nbsp;<a title="Jekyll (software)" href="https://en.wikipedia.org/wiki/Jekyll_(software)">Jekyll</a>&nbsp;static web site and blog generator and GitHub continuous integration pipelines. Each time the content source is updated, Jekyll regenerates the website and automatically serves it via GitHub Pages infrastructure.</p>
<p>As with the rest of GitHub, it includes both free and paid tiers of service, instead of being supported by&nbsp;<a title="Online advertising" href="https://en.wikipedia.org/wiki/Online_advertising">web advertising</a>. Web sites generated through this service are hosted either as&nbsp;<a title="Subdomain" href="https://en.wikipedia.org/wiki/Subdomain">subdomains</a>&nbsp;of the github.io domain, or as custom&nbsp;<a title="Domain name" href="https://en.wikipedia.org/wiki/Domain_name">domains</a>&nbsp;bought through a third-party&nbsp;<a title="Domain name registrar" href="https://en.wikipedia.org/wiki/Domain_name_registrar">domain name registrar</a>.</p>
<h3><span id="Gist" class="mw-headline">Gist</span></h3>
<p>GitHub also operates other services: a&nbsp;<a title="Pastebin" href="https://en.wikipedia.org/wiki/Pastebin">pastebin</a>-style site called&nbsp;<strong>Gist</strong>&nbsp;that is for hosting&nbsp;<a title="Snippet (programming)" href="https://en.wikipedia.org/wiki/Snippet_(programming)">code snippets</a>&nbsp;(GitHub proper is for hosting larger projects).</p>
<p>Tom Preston-Werner presented the then-new Gist feature at a punk rock Ruby conference in 2008. Gist builds on the traditional simple concept of a&nbsp;<a title="Pastebin" href="https://en.wikipedia.org/wiki/Pastebin">pastebin</a>&nbsp;by adding version control for code snippets, easy forking, and SSL encryption for private pastes. Because each "gist" has its own Git repository, multiple code snippets can be contained in a single paste and they can be pushed and pulled using Git. Further, forked code can be pushed back to the original author in the form of a patch, so gists (pastes) can become more like mini-projects.</p>

<h2><span id="Controversies" class="mw-headline">Controversies</span></h2>
<h3><span id="Harassment_allegations" class="mw-headline">Harassment allegations</span></h3>
<p>In March 2014, GitHub programmer Julie Ann Horvath alleged that founder and CEO Tom Preston-Werner and his wife Theresa engaged in a pattern of harassment against her that led to her leaving the company.&nbsp;In April 2014, GitHub released a statement denying Horvath's allegations.&nbsp;However, following an internal investigation, GitHub confirmed the claims. GitHub's CEO Chris Wanstrath wrote on the company blog, "The investigation found Tom Preston-Werner in his capacity as GitHub's CEO acted inappropriately, including confrontational conduct, disregard of workplace complaints, insensitivity to the impact of his spouse's presence in the workplace, and failure to enforce an agreement that his spouse should not work in the office."&nbsp;Preston-Werner then resigned from the company. In 2017 more allegations were made of discriminatory and unsupportive behavior at GitHub by a developer who had been recruited following a commitment by GitHub to improve its diversity and inclusivity.</p>
<h3><span id="Sanctions" class="mw-headline">Sanctions</span></h3>
<p>On July 25, 2019, a developer based in Iran wrote on Medium about how GitHub blocked his private repositories, and prohibited access to GitHub Pages.&nbsp;Soon after, GitHub confirmed that it was now blocking developers in&nbsp;<a title="Iran" href="https://en.wikipedia.org/wiki/Iran">Iran</a>,&nbsp;<a title="Crimea" href="https://en.wikipedia.org/wiki/Crimea">Crimea</a>,&nbsp;<a title="Cuba" href="https://en.wikipedia.org/wiki/Cuba">Cuba</a>,&nbsp;<a title="North Korea" href="https://en.wikipedia.org/wiki/North_Korea">North Korea</a>, and&nbsp;<a title="Syria" href="https://en.wikipedia.org/wiki/Syria">Syria</a>&nbsp;from accessing private repositories.&nbsp;However, GitHub reopened access to GitHub Pages days later, for public repositories regardless of location. It was also revealed that using GitHub while visiting sanctioned countries could result in similar action occurring on a user's account. GitHub responded to complaints and the media through a spokesperson, saying:</p>
<blockquote>
<p>"GitHub is subject to US trade control laws, and is committed to full compliance with applicable law. At the same time, GitHub's vision is to be the global platform for&nbsp;<a class="mw-redirect" title="Developer (software)" href="https://en.wikipedia.org/wiki/Developer_(software)">developer</a>&nbsp;collaboration, no matter where developers reside. As a result, we take seriously our responsibility to examine government mandates thoroughly to be certain that users and customers are not impacted beyond what is required by law. This includes keeping public repositories services, including those for open source projects, available and accessible to support personal communications involving developers in sanctioned regions."</p>
</blockquote>
<p>Developers who feel that they should not have restrictions can appeal for the removal of said restrictions, including those who only travel to, and do not reside in, those countries. GitHub has forbidden the use of&nbsp;<a title="Virtual private network" href="https://en.wikipedia.org/wiki/Virtual_private_network">VPNs</a>&nbsp;and&nbsp;<a title="Proxy server" href="https://en.wikipedia.org/wiki/Proxy_server">IP proxies</a>&nbsp;to access the site from sanctioned countries, as purchase history and IP addresses are how they flag users, among other sources.</p>
<h3><span id="Censorship" class="mw-headline">Censorship</span></h3>
<div class="hatnote navigation-not-searchable">Main article:&nbsp;<a title="Censorship of GitHub" href="https://en.wikipedia.org/wiki/Censorship_of_GitHub">Censorship of GitHub</a></div>
<p>On December 3, 2014, GitHub was blacklisted in Russia because GitHub initially refused to take down user-posted suicide manuals.&nbsp;After a day, Russia withdrew its block.</p>
<p>On December 31, 2014, GitHub was blocked in India (along with 31 other websites) over pro-<a class="mw-redirect" title="ISIS" href="https://en.wikipedia.org/wiki/ISIS">ISIS</a>&nbsp;content posted by users.&nbsp;On January 10, 2015, GitHub was unblocked.</p>
<p>On March 26, 2015, GitHub fell victim to a massive DDoS attack that lasted for more than 118 hours.&nbsp;The attack, which appeared to originate from China, primarily targeted GitHub-hosted user content describing methods of&nbsp;<a title="Internet censorship circumvention" href="https://en.wikipedia.org/wiki/Internet_censorship_circumvention">circumventing Internet censorship</a>.</p>
<p>On October 8, 2016, GitHub access was blocked by the Turkish government to prevent email leakage of a hacked account belonging to the country's Energy Minister.</p>
<p>On July 25, 2019, GitHub started blocking developers in countries facing US trading sanctions.</p>
<h3><span id="ICE_Contract" class="mw-headline">ICE Contract</span></h3>
<p>GitHub has a $200,000 contract with&nbsp;<a title="U.S. Immigration and Customs Enforcement" href="https://en.wikipedia.org/wiki/U.S._Immigration_and_Customs_Enforcement">Immigration and Customs Enforcement (ICE)</a>, for the use of their on-site product GitHub Enterprise Server. This contract was renewed in 2019, despite internal opposition from many GitHub employees. In an email sent to employees, later posted to the GitHub blog on October 9th, 2019, CEO&nbsp;<a title="Nat Friedman" href="https://en.wikipedia.org/wiki/Nat_Friedman">Nat Friedman</a>&nbsp;stated "The revenue from the purchase is less than $200,000 and not financially material for our company." He announced that GitHub had pledged to donate $500,000 to "nonprofit groups supporting immigrant communities targeted by the current administration."&nbsp;In response at least 150 GitHub employees signed an open letter re-stating their opposition to the contract, and denouncing alleged human rights abuses by ICE. As of Nov 13, 2019, 5 workers had resigned over the contract.</p>
